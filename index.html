Developer Roadmap

What must be built to make this a true production healthcare app.

Build plan
Phase 1

GitHub Pages prototype with manual calculator, education, demo public price data.

Phase 2

Backend database for hospital/insurer transparency files; search by CPT, payer, location, facility.

Phase 3

User accounts with Firebase/Supabase; save learning progress and estimates.

Phase 4

Secure AI backend for bill/EOB explanation. No API key in browser code.

Phase 5

FHIR/OAuth payer integrations for deductible status, claims, and plan benefits.

Phase 6

Prior authorization integrations with payer/EHR/clearinghouse partners.

Production warning

This prototype intentionally avoids collecting real PHI. A real version needs legal/privacy review, secure hosting, encrypted databases, consent management, and compliance controls.
